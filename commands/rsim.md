# /rsim — Recursively Self-Improving Module Health Check

Полный health check RSIm: фаза цикла, состояние проекта, здоровье скиллов, база знаний.

---

## Инструкции

Выполни все проверки молча, затем выведи один отчёт.

### Блок 1 — Фаза цикла

1. Проверь `~/.claude/reflection.md`:
   - Не существует → фаза зависит от CLAUDE.md
   - Существует, строка коррекции пуста → **REFLECT**
   - Существует, строка коррекции заполнена → **IMPROVE**
2. Проверь `CLAUDE.md` в текущей рабочей директории:
   - Не существует → **INTAKE**
   - Существует → **WORK**

### Блок 2 — Состояние проекта

Если `CLAUDE.md` существует, прочитай:
- `SESSION:` строку в QUICK CONTEXT (или `Last updated`) → вычисли дней с обновления
- Последнюю строку `LOG` (или `Session Log`) → дата + заголовок
- `IN_PROG:` строку в QUICK CONTEXT (или `What's In Progress`) → пусто или есть задачи
- Количество строк в файле → `claude_md_lines`
- Наличие секции `## QUICK CONTEXT` → есть/нет
- Наличие строки `INFRA:` → есть/нет


### Блок 3 — Здоровье скиллов

Прочитай `~/.claude/skills/index.md`:
- Посчитай общее количество скиллов (строки таблицы, не считая архив)
- Найди скиллы с `Использований = 0` → кандидаты на архив
- Найди топ-3 по количеству использований
- Проверь, есть ли скиллы со статусом отличным от `✅ Ready`
- **Посчитай скиллы со статусом `⚠️ Review`** → запиши отдельно как N_review

### Блок 4 — Синхронизация с rsim-base

Прочитай `~/.claude/skills/sync-state.json`:
- Если файл не существует → base_repo = "не настроен", даты = "никогда"
- Извлеки: `base_repo`, `last_export`, `last_sync`
- Для каждой даты вычисли дней назад (используй текущую дату сессии)

### Блок 5 — База знаний

- `~/.claude/memory/episodic/hard-problems.md` — существует? посчитай записи (строки `## [20`)
- `~/.claude/memory/episodic/best-practices.md` — существует? посчитай записи
- `~/.claude/memory/episodic/failures/` — посчитай .md файлы (не README)
- `~/.claude/memory/semantic/patterns.md` — существует? посчитай записи (строки `## `)
- `~/.claude/reflection-*.md` — посчитай файлы в `~/.claude/` (архив прошлых рефлексий)

### Блок 5б — Прогресс системы

Прочитай `~/.claude/skills/changelog.md`:
- Посчитай общее количество строк-записей формата `YYYY-MM-DD | ...` → `total_changes`
- Посчитай уникальные даты из этих строк → `total_cycles` (≈ количество improve-сессий)
- Найди скилл с наибольшим числом правок → `top_changed`
  Если `top_changed` имеет 3+ правок → отметить как нестабильный ⚠️
- Возьми последнюю запись в changelog → дата + суть изменения

Если `changelog.md` не существует → все значения = "нет данных".

### Блок 6 — Рекомендации

Генерируй рекомендации только при реальных проблемах.
Дополнительное условие:
- `top_changed` скилл имеет 3+ правок → `[SKILL] ⚠️ [name] менялся N раз — возможно нужна архитектурная переработка, не точечные правки`

Остальные условия:
Каждая рекомендация имеет тип: `[LOOP]` — цикл RSIm, `[SKILL]` — скиллы, `[KNOWLEDGE]` — база знаний.

Сортировка: сначала LOOP, потом SKILL, потом KNOWLEDGE.

Условия:
- N_review ≥ 3 → `[LOOP] КРИТИЧНО: [N] скиллов с ⚠️ Review — /improve немедленно`
- `reflection.md` существует → `[LOOP] Есть pending /improve`
- `CLAUDE.md` старше 7 дней → `[LOOP] Давно не было сессии`
- `claude_md_lines` > 150 → `[LOOP] ⚠️ CLAUDE.md too large (N lines) — run /migrate-claude-md to compress`
- QUICK CONTEXT отсутствует → `[LOOP] No QUICK CONTEXT block — run /migrate-claude-md`
- INFRA строка отсутствует → `[LOOP] No INFRA line — VPS/deploy info not indexed`
- Скиллов с 0 uses больше 5 → `[SKILL] [N] скиллов не используются — рассмотреть архивирование`
- `best-practices.md` не существует → `[KNOWLEDGE] Создать best-practices.md`
- Архив reflections кратен 10 (и > 0) → `[LOOP] Рубеж [N] сессий — рекомендован /weekly-review`
- `last_export` > 7 дней или null (и base_repo настроен) → `[LOOP] Экспортировать улучшения → /rsim-export`
- `last_sync` > 14 дней или null (и base_repo настроен) → `[LOOP] Получить обновления из base → /rsim-sync`
- base_repo = null → `[LOOP] Federation не настроена — запустить /rsim-export для настройки`

После списка — одна строка:
`Приоритетный следующий шаг: [конкретная команда из первой рекомендации]`

Если рекомендаций нет → `Всё в порядке ✓`

# /prp-create

Создай Product Requirements Prompt (PRP) для фичи: $ARGUMENTS

## Инструкции:

1. Прочитай CLAUDE.md для понимания проекта, стека и текущего состояния
2. Изучи папку `examples/` — найди похожие паттерны
3. Изучи релевантные файлы в `src/` — пойми существующую архитектуру
4. Создай файл `PRPs/[feature-slug].prp.md` на основе шаблона `~/.claude/templates/prp_base.md`

## Что должен содержать PRP:

**GOAL** — одно предложение что строим + почему + явный НЕ-scope

**SUCCESS CRITERIA** — конкретные проверяемые условия (не абстрактные)

**CONTEXT** — точные пути к файлам для изучения, ссылки на документацию, critical gotchas

**IMPLEMENTATION BLUEPRINT** — пошаговый план с указанием конкретных файлов

**VALIDATION LOOP** — команды для проверки на каждом уровне (lint → тесты → интеграция → регрессия)

## Правила:

- НЕ начинай реализацию — только создай PRP-документ
- Будь конкретен: пути к файлам, имена функций, конкретные команды
- Явно укажи что НЕ входит в scope
- Добавь CRITICAL-заметки для всего, что легко упустить
- После создания выведи краткое резюме PRP и спроси: "Утвердить и начать реализацию?"
- Если ответ `да` / `начинай` / `давай` / `утверждаю` / `go` →
  немедленно запусти `/prp-execute [путь к созданному файлу]`

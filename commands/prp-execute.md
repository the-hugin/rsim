# /prp-execute

Выполни Product Requirements Prompt: $ARGUMENTS

## Инструкции:

1. Прочитай указанный PRP-файл полностью
2. Прочитай CLAUDE.md текущего проекта
3. Запусти **Task tool** (subagent_type: `general-purpose`) — передай весь контекст в промпте
4. Дождись результата sub-агента
5. Выведи итог в основной чат и предложи обновить CLAUDE.md

## Как сформировать Task prompt:

Собери промпт из трёх частей и передай sub-агенту:

```
Ты — Implementer. Выполни PRP строго по blueprint.

Не используй контекст основного чата — работай только с тем,
что передано ниже.

=== CLAUDE.md проекта ===
[вставь полное содержимое CLAUDE.md]

=== PRP ===
[вставь полное содержимое PRP-файла]

=== Правила выполнения ===

Перед каждым шагом:
- Объяви какой шаг выполняешь
- Перечисли файлы которые будут изменены

После каждого шага:
- Запусти Validation Loop этого уровня
- Если проверка не прошла — исправь ДО перехода к следующему шагу
- Не двигайся дальше с красными тестами или TypeScript ошибками

Scope locking:
- Изменяй только файлы указанные в Blueprint
- Если нужно изменить файл вне scope — останови и сообщи об этом в итоге
- Не устанавливай зависимости не указанные в PRP

По завершении верни:
1. Статус каждого шага (✅ / ❌)
2. Список всех изменённых файлов
3. Результаты Validation Loop
4. IMPLEMENTATION NOTES для вставки в PRP
5. Любые отклонения от blueprint с объяснением
```

## После получения результата от sub-агента:

1. Заполни раздел IMPLEMENTATION NOTES в PRP-файле
2. Предложи обновить CLAUDE.md (What Works / Known Issues)
3. Предложи перенести PRP в `PRPs/completed/`

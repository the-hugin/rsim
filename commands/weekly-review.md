# /weekly-review

Агрегированный анализ накопленных reflection-файлов. Выявляет системные паттерны across projects.
Запускай раз в неделю или после 3+ завершённых reflect-циклов.

---

## Инструкции

### Шаг 1 — Собери данные

1. Найди все `~/.claude/reflection-*.md` файлы (архив прошлых рефлексий)
2. Прочитай каждый — извлеки:
   - Проект и дата
   - Секция "Трения" — что не работало
   - Секция "Ручная работа" — кандидаты на автоматизацию
   - Секция "Новые скиллы" — предложения
   - Строка коррекции пользователя
3. Прочитай `~/.claude/memory/episodic/hard-problems.md` — записи за последние 30 дней
4. Прочитай `~/.claude/skills/index.md` — счётчики использований

Если reflection-файлов меньше 2 → скажи:
```
Недостаточно данных для weekly review (найдено N файлов, нужно минимум 2).
Запусти ещё несколько сессий с /reflect.
```

### Шаг 2 — Найди повторяющиеся паттерны

Анализируй по пяти осям:

**Ось 1: Повторяющиеся трения**
Трение, которое встречается в 2+ reflection → системная проблема скилла.
Помечай: `[ПОВТОР × N]`

**Ось 2: Незакрытые кандидаты на автоматизацию**
Ручная работа, которая появлялась в 2+ reflection → давно просится в скилл.
Помечай: `[ДАВНИЙ КАНДИДАТ]`

**Ось 3: Неиспользуемые скиллы**
Скиллы с 0 uses в index.md + упоминаний нет ни в одном reflection → кандидат на архив.

**Ось 4: Траектория качества скиллов** ← Level 4
Для каждого скилла, который упоминается в 2+ reflection — составь временну́ю шкалу оценок:
```
[скилл]: reflection-1→✅  reflection-2→⚠️  reflection-3→❌
```
Интерпретация:
- Оценки улучшаются (❌→⚠️→✅) → петля работает, Improve помогает
- Оценки стабильно хорошие (✅→✅) → скилл зрелый
- Оценки ухудшаются или стоят на месте (⚠️→⚠️→⚠️) → Improve не решает проблему, нужен другой подход
- Первый раз ✅, потом ❌ → скилл был сломан после изменения, откат возможен

Помечай деградирующие скиллы: `[ДЕГРАДАЦИЯ]`
Помечай застрявшие: `[НЕ УЛУЧШАЕТСЯ × N сессий]`

Дополнительно: найди скиллы с пометкой `⚠️ Review` в index.md.
Это скиллы с ❌ из reflect, ещё не улучшенные через improve.
Помечай: `[ПРИОРИТЕТ IMPROVE]` — выноси первыми в Рекомендации.

**Ось 5: Эффективность петли** ← Level 4
Проверь: скиллы которые получили ⚠️ или ❌ в reflection N — стали лучше в reflection N+1?
- Доля улучшившихся скиллов после Improve = **коэффициент эффективности петли**
- Если < 50% улучшений → петля работает плохо, сам процесс Improve требует пересмотра
- Если ≥ 80% улучшений → петля здорова

### Шаг 3 — Выведи отчёт

```
═══════════════════════════════════════════════
  Weekly Review — [дата]
  Проанализировано: N reflection-файлов
  Период: [YYYY-MM-DD] — [YYYY-MM-DD]
═══════════════════════════════════════════════

── Системные проблемы (повторяются) ────────────
[ПОВТОР ×3] [описание трения] — встречалось в: проект1, проект2, проект3
  → Предложение: [конкретное исправление скилла]

[ПОВТОР ×2] ...

── Давние кандидаты на скиллы ──────────────────
[ДАВНИЙ КАНДИДАТ] [действие] — появлялось N раз
  → Предложение: создать скилл "[название]"

── Скиллы-кандидаты на архив ───────────────────
[название] — 0 uses, не упоминается в reflections
[или "Нет кандидатов"]

── Строки коррекции (тренды) ───────────────────
Самые частые темы в коррекциях: [тема], [тема], [тема]

── Траектория качества скиллов (Level 4) ───────
[скилл]: [оценка1] → [оценка2] → [оценка3]   [УЛУЧШАЕТСЯ / СТАБИЛЬНО / НЕ УЛУЧШАЕТСЯ / ДЕГРАДАЦИЯ]
[или "Недостаточно данных — нужно 2+ рефлексии на скилл"]

── Эффективность петли (Level 4) ───────────────
Скиллов с ⚠️/❌ в предыдущей рефлексии:   N
Из них улучшились в следующей:             M (XX%)
Коэффициент эффективности:                 [🟢 ≥80% / 🟡 50–79% / 🔴 <50%]
[или "Недостаточно данных"]

── Рекомендации ────────────────────────────────
[⚠️ ПРИОРИТЕТ IMPROVE: скиллы с ❌-историей (⚠️ Review в index.md)]
1. [конкретное действие]
2. [конкретное действие]

[GATE: если reflection-файлов ≥ 5:]
⚡ Накоплено достаточно данных для /meta-audit — внешней проверки качества петли.
   Создать скилл /meta-audit? (да / нет)

Запустить /improve для применения изменений? (да / нет)
═══════════════════════════════════════════════
```

### Шаг 4 — Если "да"

Составь синтетический reflection.md из найденных паттернов и запусти improve.
Строка коррекции = самая частая тема из коррекций пользователя.

---

## ПРАВИЛА

- Не создавай новые скиллы автоматически — только предлагай в отчёте
- Повтор считается от 2+ appearances в разных reflection-файлах (не в одном)
- Архивирование скилла — только предложение, не действие

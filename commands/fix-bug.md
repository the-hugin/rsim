# /fix-bug

Исправь баг: $ARGUMENTS

## Инструкции:

1. Прочитай CLAUDE.md — проверь Known Issues (может уже задокументировано)
2. Воспроизведи проблему — опиши симптом точно
3. Найди первопричину — не симптом, а причину
4. Предложи минимальный fix — жди подтверждения

## Debug Protocol:

```
Симптом:           [что происходит]
Ожидаемое:         [что должно быть]
Первопричина:      [почему это происходит]
Минимальный fix:   [что изменить]
Затронутые файлы:  [список]
Риски:             [что может сломаться]
```

## Pattern Analysis (перед fix):

Найди в проекте аналогичный **рабочий** код:
- Похожая функция / компонент / модуль — что у него есть, чего нет у сломанного?
- Сравни конфигурацию, зависимости, структуру данных
- Задокументируй **каждое различие** между рабочим и сломанным

## Правило трёх попыток:

Если **три фикса подряд не помогли** — стоп.
Не продолжай симптоматические правки. Вместо этого:
1. Признай что модель проблемы была неверной
2. Пересмотри архитектурные предположения
3. Опиши ситуацию пользователю и предложи пересмотр подхода

## Scope Rules:

- Исправляй только указанный баг — не рефакторь попутно
- Минимальные изменения — меньше diff, меньше регрессий
- Если fix требует большой рефакторинг — сообщи и создай отдельный PRP

## После фикса:

1. `npm run test` — убедись что существующие тесты зелёные
2. Покажи diff изменённых файлов
3. Предложи добавить тест для этого бага (регрессионный тест)

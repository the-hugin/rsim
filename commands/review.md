# /review

Сделай code review: $ARGUMENTS

## Что проверяй:

### 1. Корректность
- Логика соответствует требованиям?
- Edge cases обработаны?
- Error handling правильный?

### 2. Согласованность с проектом
- Следует паттернам из `examples/`?
- Соответствует конвенциям из CLAUDE.md?
- Не нарушает архитектурные решения из ADR?

### 3. Качество кода
- TypeScript типы корректны?
- Нет дублирования (DRY)?
- Функции делают одно дело?
- Нет магических чисел и захардкоженных значений?

### 4. Тесты
- Покрыты happy path и edge cases?
- Тесты тестируют поведение, а не реализацию?

### 5. Безопасность
- Нет секретов в коде?
- Входные данные валидируются?
- Нет уязвимостей (XSS, injection)?

## Формат вывода:

```
✅ Хорошо:    [что сделано правильно]
⚠️  Улучшить:  [что стоит изменить + почему]
❌ Проблема:  [что нужно исправить обязательно]
```

Будь конкретен: указывай файл, строку, и предлагай конкретное исправление.
